What's a Splay Tree?

A binary search tree wherein the more you search something the faster it gets to search it Accomplished by moving a searched object closer to the root position while maintaining the binary search tree. Done by using a series of moves called zigs and zags Searched object always moves 2 spaces closer to the root with each search

What's a zig and zag?

  • These are Rotations of a binary tree
    • Zig - Rotates to the right Ex. x becomes parent of y, y becomes right child of x, and x's former right child becomes y's left child
    • Zag - Rotates to the left Ex. y becomes parent of x, x becomes left child of y, and y's former left child becomes x's right child
  • Used when searched node is a child of the root

What's a zig-zig and zag-zag?

  • Combination of 2 moves, used when the searched node has a parent and a grandparent
    • Zig-Zig - two consecutive right rotations left-aligned on the grand parent then parent, done when the searched node is the left child of the parent and the parent is the left of the GP
    • Zag-Zag - two consecutive left rotations left-aligned on the grand parent then parent, done when the searched node is the right child of the parent and the parent is the right of the GP

Zag-Zig and Zig-Zag

  • The combinations used for the other cases: left child of parent but parent is the right child of GP and right child of parent but parent is left child of GP
    • Zig-Zag - LC of P and RC of GP - You Zig on the Parent then Zag on the GP
    • Zag-Zig - RC of P and LC of GP - You Zag on the Parent then Zig on the GP

Now Let's talk about Splaying

  • Splaying is using the aforementioned Zig, Zag, Zig-Zig, Zag-Zag, Zig-Zag,and Zag-Zig rotations on a node
  • You splay after a search, after inserting a new node, and before deleting a node
  • Also after each splay the splay tree will become more balanced, but note: A splay tree doesn't need to be balanced it only has to be a BST

Ex: Splaying after searching

Insertion & Deletion

Insertion

  • If BST is empty create the new node as the root and you are finished
  • If not then insert the node as you would normally in a BST
  • Proceed to splay the new node

Deletion

  • Splay target node until its at the root
  • Proceed to remove the node as you would normally in a BST

Some O(n) details and Typical applications

  • All operations on average run in O(logn) on average
  • And in worse case O(n)
